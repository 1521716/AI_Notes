# Prompt

---

## 📋 标准格式

```markdown
【角色】你是一位 [具体身份]。
【任务】请帮我 [做什么]。
【要求】
1. [约束一]
2. [约束二]
3. [约束三]
【输出格式】用 [什么形式] 呈现。
```

---

## 💡 我的理解

Prompt 是我给 AI 下达的**任务说明书**。一个差的 Prompt 会让 AI 猜我的意图，一个结构化的好 Prompt 能消除 AI 的困惑，直接给出我想要的答案。

### 四要素

标准格式包含四个关键要素：

1. **角色** - 定义 AI 的身份和专业背景
2. **任务** - 明确要完成的具体工作
3. **要求** - 列出约束条件和质量标准
4. **输出格式** - 指定结果的呈现方式

---

## 🚀 为什么重要

好的 Prompt 不是技巧，是**工程效率**。它把我的思考成本前置，换来 AI 的更高质量输出，省去反复修改提问的时间。这是我作为程序员与 AI 协作的第一项基本功。

---

## 对比：好 Prompt vs 坏 Prompt

### 1. 坏 Prompt 的特点

**缺点**：
- AI 对要求很模糊，有很多疑问（如语言选择、功能范围等）
- 疑问占据思考过程的大部分篇幅
- 生成的代码是一大段完整程序，不便于分析和定位错误

**优点**：
- 会陈述代码的大致功能（运行方式、数据存储、支持功能）
- 代码添加了部分注释
- 会按照常见需求自由发挥

### 2. 好 Prompt 的特点

**优点**：
- 思考过程篇幅增长，疑问减少
- 能明确很多要求（语言、功能、类结构、输出步骤等）
- 按步骤逐段给出代码，每段后有解释说明
- 便于理解和排错

**缺点**：
- 需要编写详细、规范的 Prompt，这本身是一个难点
