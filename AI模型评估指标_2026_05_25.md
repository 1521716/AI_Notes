# AI模型评估指标

> **一句话总结**：用混淆矩阵和四大核心KPI（准确率、精确率、召回率、F1 Score）评判模型性能，避免被数据不平衡误导。

---

## 📬 比喻：海关安检的"查违禁品"系统

| 模型概念 | 海关场景 | 垃圾邮件场景 |
|----------|----------|--------------|
| **正例 (Positive)** | 携带违禁品的包裹 | 垃圾邮件 |
| **负例 (Negative)** | 正常包裹 | 正常邮件 |

---

## 2×2 混淆矩阵：一切指标的根源

| | 预测为：违禁品 (Spam) | 预测为：正常 (Ham) |
|---|---------------------|-------------------|
| **实际是：违禁品** | **TP** ✅ 真阳性（查对了） | **FN** ❌ 假阴性（漏网之鱼） |
| **实际是：正常** | **FP** ❌ 假阳性（误伤了） | **TN** ✅ 真阴性（放对了） |

**垃圾邮件场景解释**：
- **TP**：垃圾邮件正确扔进垃圾箱
- **TN**：正常邮件正确进入收件箱
- **FP**：正常邮件被误判为垃圾（老板邮件被误拦）
- **FN**：垃圾邮件漏网进入收件箱（钓鱼邮件）

---

## 📊 四大核心指标

### 1. 准确率 (Accuracy)
**公式**：`(TP + TN) / (TP + TN + FP + FN)`  
**含义**：随机拿一个样本，判断正确的概率。  
**陷阱**：数据不平衡时完全失效！比如1000封邮件中只有10封垃圾，"全部放行"策略准确率99%，但完全没用。

### 2. 精确率 (Precision)
**公式**：`TP / (TP + FP)`  
**含义**：系统标记为"垃圾"的邮件中，真正是垃圾的比例。衡量**查准能力**。  
**例子**：标记100封垃圾，其中20封是误判，精确率 = 80/100 = 80%  
**低精确率** = 滥杀无辜，误伤太多。

### 3. 召回率 (Recall)
**公式**：`TP / (TP + FN)`  
**含义**：所有真实垃圾邮件中，被成功揪出的比例。衡量**查全能力**。  
**例子**：100封真垃圾只抓了80封，召回率 = 80/100 = 80%  
**低召回率** = 玩忽职守，漏掉太多威胁。

### 4. F1 Score
**公式**：`2 * (Precision * Recall) / (Precision + Recall)`  
**含义**：精确率和召回率的调和平均数，惩罚偏科严重的系统。  
**例子**：
- 保守系统：100%精确率，0%召回率 → F1 = 0
- 激进系统：20%精确率，100%召回率 → F1 ≈ 33%
- 平衡系统：80%精确率，80%召回率 → F1 = 80%

---

## 🧠 四种指标对比表

| 指标 | 关注点 | 海关比喻 | 代价 |
|------|--------|----------|------|
| **准确率** | 整体表现好吗？ | 大部分包裹处理正确 | 数据不平衡时无用 |
| **精确率** | 我错抓了吗？ | 扣下的包裹真的是违禁品吗？ | 漏掉一些垃圾邮件 |
| **召回率** | 我漏掉了吗？ | 所有违禁品都扣下了吗？ | 误伤正常邮件 |
| **F1 Score** | 我平衡吗？ | 不错杀也不漏抓 | 追求平衡 |

---

**核心价值**：看懂模型成绩单，避免被表面指标误导，选择真正有用的模型。
