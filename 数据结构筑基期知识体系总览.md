## 一、线性结构

### 1.1 数组 / vector ★★★

**一句话定义**：连续内存存储的同类型元素集合，支持随机访问。

**核心操作**：

- 随机访问：`arr[i]` → O(1)
- 插入/删除：`insert/erase` → O(n)
- 遍历：`for` 循环 → O(n)

**时间复杂度**：

| 操作   | 时间复杂度          |
| ---- | -------------- |
| 随机访问 | O(1)           |
| 尾部插入 | O(1) amortized |
| 中间插入 | O(n)           |

**LeetCode典型题号**：1. 两数之和、27. 移除元素

**代码文件路径**：无专门文件（STL容器基础）

***

### 1.2 链表（单向/双向） ★★★

**一句话定义**：通过指针串联的节点序列，节点可动态创建。

**核心操作**：

- 插入：`next` 指针重连 → O(1)（已知位置）
- 删除：指针跳过待删节点 → O(1)（已知位置）
- 查找：遍历链表 → O(n)

**关联标注**：

- 链表 → 可用于实现 **栈** 和 **队列**（链式存储）

**代码文件路径**：无专门文件

***

### 1.3 栈（顺序栈/链栈 | std::stack） ★★★

**一句话定义**：后进先出（LIFO）的线性表，只允许在一端操作。

**核心操作**：

- `push`：入栈 → O(1)
- `pop`：出栈 → O(1)
- `top`：查看栈顶 → O(1)

**关联标注**：

- 栈 → **递归** 的非递归实现
- 栈 → **DFS**（图的深度优先搜索）
- 栈 → **二叉树迭代遍历**（前/中/后序）

**代码文件路径**：无专门文件（STL `std::stack` 直接使用）

***

### 1.4 队列（顺序队列/循环队列/链队 | std::queue） ★★★

**一句话定义**：先进先出（FIFO）的线性表，允许在一端插入、另一端删除。

**核心操作**：

- `push`：入队 → O(1)
- `pop`：出队 → O(1)
- `front`：查看队首 → O(1)

**关联标注**：

- 队列 → **BFS**（图的广度优先搜索）
- 队列 → **树的层序遍历**

**代码文件路径**：无专门文件（STL `std::queue` 直接使用）

## 五、图算法

### 5.1 Dijkstra 单源最短路径 ★★★

**一句话定义**：在带权图中，从一个源点到所有其他顶点的最短路径算法（权重非负）。

**核心思想**：贪心策略，每次选择当前距离最短的未访问顶点

**算法步骤**：

1. 初始化：`dist[源点]=0`，其他`dist=∞`
2. 选取当前最短距离顶点`u`
3. 用`u`更新邻接点距离：`dist[v] = min(dist[v], dist[u] + w(u,v)`
4. 重复直到所有顶点被访问

**关联标注**：

- **优先队列 = Dijkstra的加速器**（用堆代替线性扫描找最小值）
- Dijkstra → **地图导航**、**网络路由**

**LeetCode典型题号**：743. 网络延迟时间、787. K 站中转内最便宜航班

**代码文件路径**：`d:\编程学习\C++学习\C++练习\Dijkstra_2026_06_01\Dijkstra_2026_06_01.cpp`

### 5.2 Prim 最小生成树 ★★★

**一句话定义**：从任意顶点开始，逐步添加不会形成环的最小权重边，直到包含所有顶点。

**核心思想**：贪心策略，类似Dijkstra但选择的是连接已选集合与未选集合的最小边

**算法步骤**：

1. 从任意顶点开始，标记为已选
2. 找到连接已选集与未选集的最小边
3. 将该边加入MST，标记新顶点
4. 重复直到所有顶点被选

**关联标注**：

- **Prim = 单源最短路径的变体**
- Prim → **电路布线**、**城市光缆铺设**

**LeetCode典型题号**：1135. 最低成本联通所有城市、1584. 连接所有点的最小费用

## 六、排序算法

### 6.1 O(n²) 排序（冒泡/选择/插入） ★★☆

**一句话定义**：时间复杂度为平方级别的简单排序算法。

| 算法   | 思路           | 稳定性   | 特点          |
| ---- | ------------ | ----- | ----------- |
| 冒泡排序 | 相邻比较，大的泡泡冒上去 | ✅ 稳定  | 最简单         |
| 选择排序 | 每次选最小/最大     | ❌ 不稳定 | 交换次数少       |
| 插入排序 | 局部有序，逐步扩展    | ✅ 稳定  | 对接近有序的序列效率高 |

**关联标注**：

- 插入排序 → **希尔排序**的基础

**代码文件路径**：无专门文件（简单回顾）

***

### 6.2 O(n log n) 排序（快排/归并/堆排） ★★★

#### 横向对比表

| 算法       | 平均         | 最坏         | 空间       | 稳定性   | 核心操作        |
| -------- | ---------- | ---------- | -------- | ----- | ----------- |
| **快速排序** | O(n log n) | O(n²)      | O(log n) | ❌ 不稳定 | partition分区 |
| **归并排序** | O(n log n) | O(n log n) | O(n)     | ✅ 稳定  | merge合并     |
| **堆排序**  | O(n log n) | O(n log n) | O(1)     | ❌ 不稳定 | swim/sink   |

#### 快速排序 ★★★

**核心**：选基准，分区，递归左右
**特点**：实际应用中常数最小，速度最快；不稳定

#### 归并排序 ★★★

**核心**：分而治之，从小区间归并到大区间
**特点**：稳定排序，适合外部排序；需要O(n)额外空间

#### 堆排序 ★★★

**核心**：构建最大堆，堆顶与末尾交换，sink调整
**特点**：原地排序，空间O(1)；不稳定

**关联标注**：

- **堆的swim/sink = 堆排序的核心操作**
- 归并排序 → **外部排序**（大数据文件分割排序）

**LeetCode典型题号**：912. 排序数组（排序算法综合）

**代码文件路径**：
