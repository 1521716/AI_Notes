# Linux学习笔记：chmod与执行权限详解（2026.06.12）

## chmod 基本用法

- **增加权限**：`chmod u+x file`（给所有者加上执行权限）
- **移除权限**：`chmod go-w file`（去掉组和其他人的写权限）
- **精确设置**：`chmod 755 file`（不管原来是什么，直接设成 755）

权限控制的是**所有者（u）、所属组（g）、其他人（o）** 这三种角色各自能对文件进行的操作。

---

## 执行权限到底是什么？为什么需要它？

这是 Linux 权限体系中最精巧的设计。**执行权限（x）对文件和对目录，含义完全不同。**

---

### 一、对于普通文件：执行权限 = "可以当程序运行"

**比喻：一份乐谱**
- `r`（读）：你能看乐谱上的音符。
- `w`（写）：你能修改乐谱。
- **`x`（执行）** ：你能让乐队照着这份乐谱演奏！没有 `x`，乐谱就只是一张纸，有 `x` 它才能"活"起来。

**具体来说：**

| 文件类型 | 有 x 权限 | 没有 x 权限 |
|---------|----------|------------|
| **脚本文件**（.sh） | 可以直接 `./script.sh` 运行 | 只能用 `bash script.sh` 间接执行 |
| **二进制程序**（/bin/ls） | 可以直接运行 | 无法运行，系统不认它是可执行文件 |
| **普通文本/图片** | 没意义，文件内容不是指令，无法"执行" | 正常，这类文件本就不该有 x |

**举例验证：**

```bash
echo "echo Hello" > test.sh
chmod 644 test.sh    # 只有 rw-r--r--，没有 x
./test.sh            # 报错：Permission denied
chmod 755 test.sh    # 加上 x
./test.sh            # 成功输出 Hello
```

**为什么要分开读写和执行？**

这是安全设计。你可以读一个配置文件（如 `/etc/passwd`），但不能执行它，因为它不是程序。如果误给普通文件加了 `x`，系统不会阻止你（它不知道这文件是不是真的程序），但如果你试图执行一个纯文本，shell 会尝试解析它，通常会报错。这正是"最小权限原则"的体现——不该执行的，就别给执行权限。

---

### 二、对于目录：执行权限 = "可以进入这个目录"

**这是新手最容易困惑的地方！** 目录的 `x` 权限，不是"执行目录"，而是**"能否 cd 进去"**。

**比喻：一个房间**
- `r`（读）：你能透过窗户看房间里有什么（`ls` 列出文件名）。
- `w`（写）：你能在房间里放东西或拿走东西（创建、删除文件）。
- **`x`（执行）** ：你**能推开门走进这个房间**（`cd` 进去）！

如果没有 `x`，你连门都进不去，更别说看里面有什么了。

**举例验证：**

```bash
mkdir testdir
touch testdir/inside.txt
chmod 644 testdir     # rw-r--r--，没有 x
cd testdir            # 报错：Permission denied！
ls testdir            # 可以列出文件名，但会显示权限错误
```

**一个经典实验，让你彻底明白：**

```bash
mkdir demo
echo "secret" > demo/file.txt

# 实验1：有 x 但没有 r
chmod 300 demo        # -wx------，有 w 和 x，没有 r
cd demo               # 可以进入！
ls demo               # 拒绝显示文件名（没有 r）
# 但如果你知道文件名，可以直接读取：
cat demo/file.txt     # 成功！因为你对文件有 r 权限

# 实验2：有 r 但没有 x
chmod 600 demo        # rw-------，有 r 没有 x
cd demo               # 无法进入！
ls demo               # 可以列出文件名，但每个文件后面显示 "Permission denied"
cat demo/file.txt     # 直接读也被拒绝（因为要经过目录的 x）
```

**结论：目录的 `x` 是进入目录的"通行证"，`r` 是"清单"，`w` 是"修改权"。** 通常目录都是 `755`（rwxr-xr-x），保证能进去，别人也能进来看列表。

---

## 一句话总结

| 权限 | 对文件 | 对目录 |
|------|--------|--------|
| `r` | 读文件内容 | 列出目录内容（ls） |
| `w` | 修改文件内容 | 在目录中创建/删除文件 |
| **`x`** | **作为程序执行** | **进入目录（cd）** |

---

## 常用命令总结

```bash
chmod u+x script.sh   # 给所有者加执行权限
chmod 755 myapp       # 设置为 rwxr-xr-x
chmod 644 config.ini  # 设置为 rw-r--r--
chmod go-rwx dir/     # 移除组和其他人的所有权限
chmod -R 755 project/ # 递归修改目录及其内容的权限
```
