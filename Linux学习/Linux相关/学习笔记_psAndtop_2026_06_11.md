# Linux学习笔记：ps与top命令详解（2026.06.11）

## 核心区别概览

1. **`ps` 是执行瞬间的快照**：就像对当前系统进程拍了一张高清照片，定格在你敲回车的那一毫秒，然后打印出来，任务就结束了。

2. **`top` 是实时更新的监控面板**：它像一个持续刷新的仪表盘，默认每3秒更新一次数据，你可以在里面进行操作，直到手动退出。

3. **`top` 默认确实按CPU使用率排序**：一进去，最吃CPU的进程就高高挂在最上面，所以叫"top"。

---

## 更全面的区别对比

我们用一张表把它们看透：

| 对比维度 | `ps` | `top` |
|----------|------|-------|
| **工作模式** | 快照（Snapshot）：拍完即结束，不会自动刷新。 | 实时监控（Monitor）：持续运行，动态刷新。 |
| **默认行为** | 默认只显示当前终端、当前用户的少量进程。 | 默认显示系统所有用户的进程概况和资源总览。 |
| **常用用法** | `ps aux` 看所有进程全貌；`ps aux \| grep` 精准查找。 | 直接 `top` 启动，再配合交互按键（如 `P`、`M`、`k`）动态操作。 |
| **更新方式** | 一次性输出，管道或重定向拿到的是静态数据。 | 实时刷新，按 `-d` 可调间隔，如 `top -d 1`。 |
| **排序逻辑** | 不加 `--sort` 按 PID 排，可手动指定排序字段。 | 默认按 CPU 使用率降序，按 `M` 切内存，按 `T` 切时间。 |
| **交互能力** | 无，一次性退出。 | 强交互：可在界面内排序、杀进程、过滤用户。 |
| **资源消耗** | 极低，抓一下数据就释放。 | 持续占用少量 CPU 刷新，但一般可忽略。 |
| **典型场景** | 脚本取进程列表、管道分析、快速核实进程在不在。 | 日常巡检、排查性能瓶颈、处理失控进程。 |

## 比喻强化记忆

- **`ps aux`** 是保安**每月拍一张**园区人员合影，存档到文件夹里，想查谁拿照片用放大镜（`grep`）找。
- **`top`** 是保安**坐在监控室里**盯着一整面墙的实时屏幕，画面一直在动，看见可疑人物直接就按报警器（`k` 杀进程）。

---

## 一个你可能没注意的关键点

`ps aux` 显示的 CPU 使用率，是进程**从启动到现在的平均占用**；而 `top` 里的 CPU 是**瞬时占用**，每一帧都可能不同。所以有时 `top` 里看到一个进程飙到 90%，但 `ps` 里它可能只有 5%。这是两者数据采集方式决定的，排查瞬间飙高的问题必须用 `top`。

## 常用命令总结

### ps 常用命令
```bash
ps aux              # 查看所有进程的详细信息
ps aux | grep nginx # 查找特定进程
ps -ef              # 另一种查看进程的方式，显示父进程PID
```

### top 常用交互按键
```bash
top                 # 启动top
top -d 1            # 每秒刷新一次
```

进入top后：
- **P**：按CPU使用率排序（默认）
- **M**：按内存使用率排序
- **T**：按CPU时间排序
- **k**：杀死进程（需要输入PID）
- **q**：退出top
