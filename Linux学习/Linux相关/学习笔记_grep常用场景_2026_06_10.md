# Linux学习笔记：grep常用场景实战（2026.06.10）

好的同学，基于你已掌握的 `-i`、`-v`、`-n`、`-c`、`-r` 这五个选项，我为你设计了5个**真实运维场景**，每个都配有命令和解释。你现在就可以在自己的终端里试一试。

---

## 场景1：在配置文件中快速找到某个参数的行号

**场景描述：**

你正在调整 SSH 配置，想立刻知道 `Port` 这个参数在 `/etc/ssh/sshd_config` 的第几行，并且忽略大小写，防止文件里写成 `port` 导致搜不到。

**命令：**

```bash
grep -in "port" /etc/ssh/sshd_config
```

**选项拆解：**

- `-i`：忽略大小写，`Port`、`PORT`、`port` 都能匹配。
- `-n`：显示行号，直接定位。

---

## 场景2：统计日志中错误出现次数，快速判断严重程度

**场景描述：**

服务器上的 `/var/log/syslog`（或 `/var/log/messages`）文件很大，你不想翻内容，只想立刻知道里面有多少行包含 `error`，以便评估是否需要紧急处理。

**命令：**

```bash
grep -ci "error" /var/log/syslog
```

**选项拆解：**

- `-c`：只输出匹配的行数，不显示具体内容。
- `-i`：忽略大小写，同时统计 `Error`、`ERROR` 等。

---

## 场景3：反向过滤，查看日志中非正常信息之外的"安静"部分

**场景描述：**

你在浏览日志时，发现满屏都是 `debug` 调试信息，想暂时把这些行**全部忽略**，只看没有 `debug` 的内容，方便发现其他级别的日志。

**命令：**

```bash
grep -v "debug" /var/log/syslog | tail -20
```

**选项拆解：**

- `-v`：反向匹配，输出不包含 `debug` 的行。
- 结合 `tail -20` 只看最后 20 行，避免刷屏。

---

## 场景4：递归搜索项目目录，找出包含"TODO"注释的所有文件

**场景描述：**

你有一个代码目录 `~/myblog`，你想知道哪些源文件里还留有 `TODO` 标记，但不需要看具体哪一行，只要文件名列表即可。

**命令：**

```bash
grep -ri "todo" ~/myblog
```

**选项拆解：**

- `-r`：递归搜索目录下所有文件。
- `-i`：忽略大小写，防止 `TODO`、`Todo` 被遗漏。

（注：你已学过 `-r`，这里没用 `-l` 因为我们仅要求用学过的选项；但用 `-r` 会显示匹配行，也算完成任务。如果只想显示文件名，需要 `-l`，但不在你已学选项内，所以先用 `-r`。实际中加上 `-l` 会更清爽。）

---

## 场景5：实时监控日志，只关注包含"fail"或"denied"的行

**场景描述：**

你想实时查看系统认证日志 `/var/log/auth.log`（或 `secure`），当有人尝试登录失败时立刻看到，并且忽略大小写，同时显示行号（虽然实时流下行号可能无意义，但作为练习用途）。

**命令：**

```bash
tail -f /var/log/auth.log | grep -in "fail\|denied"
```

**选项拆解：**

- `tail -f` 实时输出新增日志行。
- `grep -i`：忽略大小写，匹配 `Fail`、`FAIL` 等。
- `grep -n`：显示行号（在流中会累加行号）。
- 搜索模式 `"fail\|denied"` 表示匹配 `fail` 或 `denied`（使用了基本正则的或逻辑，不算新选项，是搜索模式本身的写法）。

（提醒：如果只想用固定字符串，可以用两个 `grep` 结合，但这里用了正则的 `\|`，算延伸。纯用已学选项的话，可以写成 `tail -f /var/log/auth.log | grep -i "fail"` 或结合管道再次过滤，不影响练习。）

---

## 场景总结

这5个场景覆盖了日常排查、统计、过滤、递归查找和实时监控：

| 场景 | 核心需求 | 使用的选项组合 |
|------|---------|---------------|
| 场景1 | 定位行号 + 忽略大小写 | `-in` |
| 场景2 | 统计次数 + 忽略大小写 | `-ci` |
| 场景3 | 反向过滤 + 限制输出 | `-v` + `tail` |
| 场景4 | 递归搜索目录 | `-ri` |
| 场景5 | 实时监控 + 多模式匹配 | `tail -f` + `grep -in` |
