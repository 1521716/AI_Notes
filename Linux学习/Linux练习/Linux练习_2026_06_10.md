## 作业批改与分析

### 练习1：系统体检

**你的答案：**

```bash
find /etc -name *.conf | sort
find /etc -name *.conf | wc -l | tail -1
tail -15 /var/log/syslog > system_report.txt
```

🔍 **逐条分析：**

1. **第一条**：`find /etc -name *.conf | sort`
   ⚠️ `*.conf` 没有用引号，在 shell 里会被**当前目录**的文件扩展。如果当前目录有 `a.conf`、`b.conf`，`*.conf` 会展开成 `a.conf b.conf`，导致 `find` 的 `-name` 参数错误。**一定要用引号**：`find /etc -name "*.conf"`。其余逻辑（查找 + 排序）正确。

2. **第二条**：`find /etc -name *.conf | wc -l | tail -1`
   ⚠️ 同样有引号问题。
   ⚠️ `wc -l` 会输出一个数字，只有一行，再用 `tail -1` 是多余的。直接 `find ... | wc -l` 即可。

3. **第三条**：`tail -15 /var/log/syslog > system_report.txt`
   ❌ 题目要求**追加**到 `system_report.txt`，你用 `>` 会覆盖原内容。应该用 `>>`。
   ✅ 正确：`tail -15 /var/log/syslog >> system_report.txt`

✅ **修正后的命令序列：**

```bash
find /etc -name "*.conf" | sort
find /etc -name "*.conf" | wc -l
tail -15 /var/log/syslog >> system_report.txt
```

### 练习3：日志分析

**你的答案：**

```bash
echo -e "192.168.1.1 - - [10/Jun/2024:10:00:00] "GET /index.html" 200\n192.168.1.1 - - [10/Jun/2024:10:01:00] "POST /login" 401\n192.168.1.1 - - [10/Jun/2024:10:02:00] "GET /about" 200" > access.log
grep -n "200" access.log
cut " " 1 access.log | sort | uniq -n | sort -rn
```

🔍 **分析：**

1. **生成日志文件**：
   ❌ 你的 `echo` 命令里，`"GET /index.html"` 的引号和外面的引号冲突，会导致语法错误。应该用单引号包裹整个字符串，或者用转义。另外，你给出的三行 IP 其实都是 `192.168.1.1`，而题目要求第二行是 `192.168.1.2`，虽然不影响练习，但最好按题目来。
   ✅ 正确写法（用单引号，避免转义）：
   ```bash
   echo '192.168.1.1 - - [10/Jun/2024:10:00:00] "GET /index.html" 200' > access.log
   echo '192.168.1.2 - - [10/Jun/2024:10:01:00] "POST /login" 401' >> access.log
   echo '192.168.1.1 - - [10/Jun/2024:10:02:00] "GET /about" 200' >> access.log
   ```

2. **grep 查找**：`grep -n "200" access.log`
   ✅ 正确，会显示包含 200 的行及其行号。

3. **cut、sort、uniq 组合**：`cut " " 1 access.log | sort | uniq -n | sort -rn`
   ❌ `cut` 的选项应该是 `-d` 来指定分隔符，格式为 `cut -d " " -f 1`。你写成了 `cut " " 1`，这是错误的。
   ❌ `uniq -n` 是错误的，`uniq` 没有 `-n` 选项，应该用 `uniq -c` 统计次数。
   ❌ 最后的 `sort -rn` 没问题，按数值降序排列。

✅ **修正后的统计管道：**

```bash
cut -d " " -f 1 access.log | sort | uniq -c | sort -nr
```

输出会显示类似：

```
      2 192.168.1.1
      1 192.168.1.2
```

### 练习5：目录导航与信息提取

**你的答案：**

```bash
su root
cd /var/log
ls -Slr | head -5
cd
find -size 0 | sort -r > "~/empty_files.txt"
less "~/empty_files.txt"
G
q
```

🔍 **逐条检查：**

1. **`su root` 与 `cd /var/log`**
   ✅ 思路对，因为 `/var/log` 通常需要 root 权限。但在实际环境中可能不知道 root 密码。题目提示“如果权限不够，用 ~ 家目录代替”，这里你用了 `su`，也算一种解决方案。注意 `su root` 后当前目录可能变化，再 `cd /var/log` 明确，没问题。

2. **`ls -Slr | head -5`**
   ❌ **排序方向反了！** `-S` 按大小从大到小排，再加 `-r` 反转，变成了**从小到大**，这样 `head -5` 取到的是最小的 5 个文件。题目要求“文件大小排前 5 的文件”，通常指**最大的 5 个**。
   ✅ 正确写法：`ls -Sl | head -5` 或 `ls -lS | head -5`（注意第一行可能是 `total`，但这里不纠结）。
   ➜ **修正**：去掉 `-r`。

3. **`cd` 回家目录，`find -size 0`**
   ✅ `find` 不指定路径默认搜索当前目录，此时已在家目录，符合“在家目录下查找”。查找空文件的条件 `-size 0` 正确。

4. **`sort -r > "~/empty_files.txt"`**
   ❌ **引号导致路径错误！** 在双引号里，`~` 不会被展开成家目录，而是被当作普通字符。所以 `"~/empty_files.txt"` 会在**当前目录**下创建一个文件名就是 `~/empty_files.txt` 的文件，而不是到家目录下创建 `empty_files.txt`。
   ✅ 正确写法：`> ~/empty_files.txt`（不加引号），或者 `> "$HOME/empty_files.txt"`。
   ➜ **修正**：把 `"~/empty_files.txt"` 换成 `~/empty_files.txt`。

5. **`less "~/empty_files.txt"`**
   ❌ 同样的引号问题，会尝试打开当前目录下名为 `~/empty_files.txt` 的文件，但该文件不存在（因为上一步也写到了错误路径），所以 `less` 会报错。
   ✅ 正确写法：`less ~/empty_files.txt`。

6. **`G` 和 `q`**
   ✅ 如果在 `less` 里成功打开文件，`G` 跳转到末尾，`q` 退出。操作逻辑完全正确。

✅ **修正后的完整序列：**

```bash
# 如果权限不够，就用家目录练习，这里假设有权限或已 su
cd /var/log
ls -Sl | head -5           # 最大的前5个文件
cd ~                       # 或直接 cd
find . -size 0 | sort -r > ~/empty_files.txt
less ~/empty_files.txt
# 进入 less 后按 G 再按 q
```

（注意 `find .` 显式指定当前目录更清晰，你的 `find -size 0` 也可以，因为缺省就是当前目录。）

## 总结与建议

- **引号里的特殊字符**：`~`、`$` 等在双引号中大多保留扩展，但 `~` 是例外，双引号会阻止波浪号展开。记住：路径不要随便加引号，除非有空格或特殊字符，那时用 `"$HOME"` 代替 `~`。
- **排序方向**：`ls -S` 默认大在前，直接 `head` 即可取最大，无需 `-r` 反转。养成习惯，先确认默认排序顺序。
- **权限问题**：如果没有 root 密码，`su root` 会失败。实际考试或练习中，如果提示“权限不够则用家目录”，就直接用家目录，不用强行提权。
