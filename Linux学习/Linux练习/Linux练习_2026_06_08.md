### 第2题

**你的答案：**

```bash
mkdir -p ./myblog/css ./myblog/js /myblog/posts
ls -d myblog/css myblog/js myblog/posts
```

❌ **第一个命令有严重路径错误。**

你写的 `/myblog/posts` 是以 `/` 开头的**绝对路径**，这会在系统**根目录**下创建 `myblog/posts`，而不是在你的家目录下。普通用户对根目录没有写权限，这条命令会直接报错 `Permission denied`，导致整个 `mkdir` 失败（就算你用 `sudo`，也是在根下建，完全违背题意）。

✅ **正确写法：** 全部使用相对路径（前面没有 `/`），或统一加 `./` 前缀：

```bash
mkdir -p myblog/css myblog/js myblog/posts
# 或
mkdir -p ./myblog/css ./myblog/js ./myblog/posts
```

🔍 **验证命令改进：** 你的 `ls -d myblog/css myblog/js myblog/posts` 只能看这三个子目录自身存在与否，但题目要求“验证 `myblog` 目录下的结构是否已建好（只看一级子目录的名字）”，最直接的方式是 `ls myblog` 或用通配符 `ls -d myblog/*/`。

```bash
ls myblog          # 输出 css  js  posts
```

### 第3题

**你的操作序列：**

```bash
cd myblog/posts
touch first.md second.md third.md
cd ..
rmdir posts                     # 失败，因为非空
cd -                            # 回到 posts
rm -v first.md second.md third.md

```

✅ **逻辑思路正确**，你确实在用 `rmdir` 的“非空拒绝”特性来安全删除，最后也达到了目的。

⚠️ **但操作过于繁琐，且有一个隐患：** 最后你用 `rmdir posts` 时，虽然成功删除了空目录，但你在第5题会尝试重命名 `posts`，那时它已经不存在了——这直接导致第5题出错。题目是连贯的情境，你删掉了后面还要用的目录，属于“过度清理”。

🔧 **更优雅的做法（保留目录给后续题用）：**

题目要求“安全地删除空的 `posts` 目录……如果有文件残留则拒绝删除”。你可以先尝试删除，它报错后你清理文件，**但保留目录名给第5题的重命名用**？不对，第5题需要将 `posts` 重命名为 `articles`，所以 `posts` 目录必须存在。你这里如果真要删，后面的题就做不了；更合理的解读是：题目可能希望你体验 `rmdir` 的安全特性，但最终不是真删，或者你应在第5题前恢复它。不过题目没有要求恢复，我们就当是个练习，但你要意识到这个矛盾。

如果只是为了练习 `rmdir`，可以单独建一个临时空目录来演示。本题建议只体验 `rmdir` 拒绝删除的行为，不真的删：

```bash
cd myblog
rmdir posts                     # 报错，提醒你非空
# 如果需要强行清空 posts 目录内容，但题目没说，就别动
```

或者你保留 posts，等第5题重命名用。**所以你的操作序列没有顺应后续题目的状态连续性。** 以后做连续场景题，要注意“前面做的改动不能破坏后面的需求”。

### 第5题

**你的操作序列：**

```bash
mv myblog "myblog-v2"
cd "myblog-v2"
mv posts articles
cd
ls -lhd "myblog-v2" "myblog-backup"
```

❌ **第二句 `mv posts articles` 会失败！**

因为你在第3题已经把 `posts` 目录删除了，`myblog-v2` 下根本不存在 `posts` 目录，所以无法重命名。你执行时一定会看到错误：

```
mv: cannot stat 'posts': No such file or directory
```

✅ **正确逻辑**（基于保留 posts 的情况）：

如果你在第3题只尝试 `rmdir` 但没真正删掉 posts，那么这里就能成功。正确的连贯操作应是：

```bash
mv myblog myblog-v2
cd myblog-v2
mv posts articles
cd
ls -lhd myblog-v2 myblog-backup
```

你的第一句和第三、四句都是对的，问题出在第二句的前提不存在。
