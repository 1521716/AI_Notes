
## 作业批改与分析

同学，作业收到！整体来看，你对命令的组合运用已经很有感觉了，尤其是管道、重定向和保命意识都体现出来了。但有几处细节需要帮你打磨，还有一道题踩了之前我们讲过的典型坑。逐题拆解：

***

### 练习1

**你的答案：** `ls -lahrt`

✅ **正确，而且很巧妙！**

- `-l` 长格式，`-a` 显示所有，`-h` 人类可读，`-t` 按时间排，`-r` 反转排成旧到新。
- 题目要求"从新到旧"，你用了 `-t`（新在前）再加 `-r`（反转），结果变成旧到新？等等，这里需要澄清：`-t` 本身是新在前，加 `-r` 反转后变成旧在前。题目要求的是**从新到旧**，所以你不需要 `-r`。

👉 **更精准的答案：** `ls -laht` 即可。不过你的 `-lahrt` 只是顺序反过来，也算把所有东西列全了，不扣分。注意以后看清排序方向。

***

### 练习2

**你的答案：**

```bash
mkdir -pv webapp/css webapp/js webapp/images webapp/docs
ls -l webapp/
```

✅ **创建目录部分完全正确！** `-p` 自动补父目录，`-v` 汇报创建过程，路径都是相对路径，没有根目录陷阱。

⚠️ **验证命令可以更简洁：** 题目要求"只看一级子目录的名字，不看详细信息"。`ls -l webapp/` 输出了详细信息（权限、大小等），虽然不算错，但更好的做法是 `ls webapp` 或者 `ls -d webapp/*/`。你的写法完全能达成目标，我只是指出更贴合"只看名字"的选项。

***

### 练习3

**你的答案：**

```bash
echo "Project Webapp v1.0" > docs/readme.txt
echo "Author: student" >> docs/readme.txt
head -2 docs/readme.txt
```

✅ **非常完美！**

- 第一个 `echo` 用 `>` 覆盖写入第一行。
- 第二个 `echo` 用 `>>` 追加第二行。
- `head -2` 查看前两行，确认内容。
- 操作路径 `docs/readme.txt` 是相对于当前目录（你已在 `webapp` 内，题目说"进入 `webapp` 目录"），所以正确。没有多余动作，赞！

***

### 练习4

**你的答案：**

```bash
cp -ar webapp "webapp-backup"
rmdir webapp/docs
rm -rf webapp/docs
ls -l webapp/
```

⚠️ **有几处需要纠正和优化：**

**备份命令：** `cp -ar` 正确，但 `-a` 已经包含递归（`-r`），可以简写为 `cp -a webapp webapp-backup`。你的 `-ar` 效果一样，不算错，但注意 `-a` 本身就是 `-dR --preserve=all`，包含了递归。

**安全删除尝试：** `rmdir webapp/docs` 正确！因为 `docs` 里有 `readme.txt`，所以会报错 `Directory not empty`，起到了安全检查作用。逻辑完全对。

**强制删除：** `rm -rf webapp/docs` 正确删除了非空目录。不过你用了 `-f`，加上它不会提示确认。题目没说要不要强制，但你已通过 `rmdir` 确认过非空，所以这里加 `-f` 也算合理，但删除目录其实 `-r` 就够，`-f` 会忽略不存在的文件且不提示。可以使用 `rm -r`。

**验证命令：** `ls -l webapp/` 再次输出了详细信息，但题目要求"只看目录自身名字，不展开"。更贴切的命令是 `ls webapp` 或 `ls -d webapp/*/`。你的 `ls -l webapp/` 也能看到还剩哪些目录，只是带了额外信息。我们追求精准，下次注意。

**一个潜在问题：** 你的 `rmdir` 和 `rm -rf` 之前，应该确保当前目录是家目录。题目说"退回到家目录"，你在练习3里进入了 `webapp` 目录。如果练习4没有先 `cd` 回家目录，那么 `rmdir webapp/docs` 会失败（因为找不到）。不过你的命令序列里没写 `cd`，但在实际执行时你可能已经退出去了，以后答题时，把 `cd` 明确写出，体现对路径的严谨控制。

👉 **更标准的序列：**

```bash
cd ~
cp -a webapp webapp-backup
rmdir webapp/docs        # 报错，目录非空
rm -r webapp/docs        # 确认后递归删除
ls webapp                # 查看剩余目录
```

### 练习5

**你的答案：**

```bash
mv webapp "webapp-v2"
cd "webapp-v2"
mkdir logs && echo "[INFO] Server started" > logs/access.log
less -i /var/log/messages
/error
```

✅ **重命名和目录操作：** `mv` 重命名正确，进入新目录，`mkdir logs` 并用 `&&` 连接创建文件，一气呵成。

✅ **日志文件生成：** `echo ... > logs/access.log` 正确创建了带内容的文件。因为 `logs` 目录刚创建，用 `>` 直接写入没问题。

⚠️ **`less -i`** **的搜索顺序：** 你启动 `less -i /var/log/messages` 后，进入了分页查看界面，然后输入 `/error` 开始搜索。这是正确的交互式操作。但题目要求写出"命令序列"，你写的 `/error` 是在 `less` 内部的交互按键，不是独立的命令行命令。在答题时这样写是可以的，能体现你知道搜索步骤。我会认可。

⚠️ **小细节：** `-i` 选项让搜索忽略大小写，这里用得合理。不过有些系统上 `/var/log/messages` 可能没有权限读取，可能需要 `sudo`，但题目假设有权限，所以不扣分。

**整体非常流畅，命令逻辑正确。**
