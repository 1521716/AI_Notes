# Context Window（上下文窗口）

## 定义

上下文窗口：大模型一次能够记住的聊天内容+文件的最大容量（单位为Token）

它决定了模型记忆的短暂性，窗口外的东西像黑板上的字被擦掉一样永远丢失。

更大窗口是趋势，但总伴随着成本、速度的考量。

而我们用户，也可以学着在有限的"记事本"上，做最高效的信息管理大师。

---

## ⚠️ 重要现象：Lost in the Middle（迷失在中间）

模型对开头（首因效应）和结尾（近因效应）记得最清楚，中间部分成了**记忆洼地**。

### Prompt 写作技巧

根据这一现象，写 Prompt 时可以采用以下策略：

1. **开头定调**：在 Prompt 的一开始，就用一两句高度浓缩的话，说清楚任务、角色、关键约束。这就像书的前言，模型最先读到，牢牢记住。

2. **主体内容放中间**：把大段参考资料、背景文档放在中段。这里允许它被"遗忘"一些细节，但框架要在此展开。

3. **结尾点睛**：在所有资料之后，用最后一段重复你的核心要求，特别是格式、禁止事项、质量标准这些容易在这种效应里被模糊掉的规定。

### 示例

让它总结一篇长报告，别只把报告扔进去然后说"帮我总结"。可以试试这样：

```markdown
【开头】你是一位资深分析师，请用三百字以内总结下方报告，突出三个关键风险。

（此处粘贴长篇报告正文……）

【结尾】再次提醒：你的角色是资深分析师，输出不超过三百字，必须列出三个关键风险，用数字编号。
```
