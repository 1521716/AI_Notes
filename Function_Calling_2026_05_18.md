# Function Calling（函数调用）

> **一句话总结**：Function Calling是把用户模糊的自然语言意图，翻译成明确的结构化程序调用的翻译过程。

---

## 🎯 核心原理

Function Calling 让大模型学会了一种新的"输出模式"：

1. **理解意图**：模型理解用户的自然语言需求
2. **选择工具**：将意图映射到预定义的函数上
3. **提取参数**：从用户句子中提取关键信息作为函数参数
4. **输出指令**：返回结构化的JSON指令（如API调用参数）

## 🤝 协作流程

```
用户 → 自然语言需求 → 模型 → JSON指令 → 应用程序 → 执行函数 → 结果 → 用户
```

### 角色分工

| 角色 | 职责 |
|------|------|
| **模型** | 翻译意图，生成调用指令 |
| **应用程序** | 真正执行函数的"手" |

## 💡 关键特点

- **模型不执行**：模型本身不具备执行能力，它只是输出指令
- **结构化输出**：输出给机器执行的API调用指令，而非给人看的对话
- **参数提取**：自动从自然语言中提取关键信息

---

**核心价值**：让AI从"对话者"升级为"工具使用者"，是构建AI Agent的关键技术。
