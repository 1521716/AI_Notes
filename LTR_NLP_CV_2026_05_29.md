# LTR vs. NLP vs. CV（排序学习 vs. 自然语言处理 vs. 计算机视觉）

> **一句话总结**：NLP是"耳朵/嘴巴"，CV是"眼睛"，LTR是"大脑"——那个决定"此时此刻，什么最好"的核心决策引擎。

---

## 🎯 比喻：建造"智慧商场"

| 角色 | 负责系统 | 处理对象 |
|------|----------|----------|
| **NLP工程师** | 语言交流系统（迎宾机器人、客服自动回复） | 文本 |
| **CV工程师** | 视觉感知系统（监控识别、虚拟试衣） | 图像/视频 |
| **LTR工程师** | 核心决策大脑（推荐、排序、广告调度） | 人-物交互信号 |

**NLP和CV提供"眼睛"和"耳朵"，LTR在建造"大脑"**。

---

## 🆚 LTR vs. NLP vs. CV 核心区别

| 维度 | 排序学习 (LTR) | 自然语言处理 (NLP) | 计算机视觉 (CV) |
|:---|:---|:---|:---|
| **处理对象** | 人-物交互信号（点击、评分、购买） | 字-词序列（句子、段落、文档） | 像素矩阵（图片、视频帧） |
| **核心问题** | 排序 (Ranking) | 理解/生成 (Understanding/Generation) | 感知/重建 (Perception/Reconstruction) |
| **典型数据** | 搜索日志、点击流、用户行为序列 | 文本语料库（书籍、网页、对话） | 图像/视频数据集 |
| **典型模型** | 双塔模型、GBDT、LambdaMART | Transformer (BERT, GPT)、RNN/LSTM | CNN, ViT, ResNet |
| **核心指标** | AUC, NDCG, MAP | BLEU, ROUGE, Perplexity | mAP, IoU, Accuracy |
| **系统角色** | **核心决策者（主脑）** | 特征提取器（耳朵/嘴巴） | 特征提取器（眼睛） |

## 🛠️ 实战案例：拍立淘（拍照搜同款）

```
用户拍照T恤
    │
    ├─▶ CV模块：输出图片向量 [0.8, -0.2, 0.5, ...]（圆领、条纹、短袖）
    │
    ├─▶ NLP模块：用户偏好向量（ins风、韩版）
    │
    └─▶ LTR排序模型：综合所有特征，输出Top-100商品列表
```

**NLP和CV输出"原材料"，LTR是总厨，把所有原料烹成"满汉全席"**。

## 📚 学习路径地图

### 基础必修（已掌握）
- Pointwise / Pairwise / Listwise 三大范式
- AUC, NDCG, MAP 评估指标
- Pairwise Ranking Loss 原理

### 工业级进阶
| 方向 | 内容 |
|------|------|
| **经典LTR模型** | GBDT家族（LambdaMART）、FM/FFM |
| **深度学习排序** | DeepFM, DCN, 双塔模型（User Tower + Item Tower） |
| **多目标排序** | 同时优化CTR、CVR、用户时长、内容多样性 |

### 辅助技能
| 方向 | 重点 |
|------|------|
| **NLP** | 文本Embedding、语义相似度计算 |
| **CV** | 多模态Embedding（图片/视频特征向量） |

---

**核心价值**：LTR工程师 = 以排序为核心、左手NLP右手CV的决策工程师。
