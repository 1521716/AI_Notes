# FIM（Fill-In-the-Middle，中间填充）

> 核心思想：**根据上文+下文，预测中间缺失的部分**。不是续写，而是高难度的"完形填空"。

---

## 🤔 一个比喻：接龙作家 vs 上帝视角作家

| 类型 | 方式 |
|------|------|
| 接龙作家 | 只看到开头，顺着往下写 |
| 上帝视角作家 | 看到开头+结尾，精准填中间 |

**例子**：
- 开头："今天早上起床后，发现窗外___"
- 结尾："……所以我决定带伞出门"
- 答案：**下雨了**

---

## 🕵️ FIM训练原理

### 普通训练：单行道续写
只给"前缀+后缀"，让模型预测中间缺失的内容。

### FIM训练：三段式拼接
训练时故意把代码分成三部分：
- `<PRE>` 前缀：上文线索
- `<SUF>` 后缀：下文线索
- `<MID>` 中间：被挖掉需要预测的部分

**示例**：
```cpp
// 原始代码
int main() {
    std::vector<int> data = {1, 2, 3, 4, 5};
    int sum = 0;
    for (int val : data) {
        sum += val;  // ← MID
    }
    return sum;
}

// FIM训练输入
<PRE> int main() { ... for (int val : data) { <SUF> } return sum; <MID>
```

### ⚠️ 重要区别：训练 vs 推理

| 阶段 | 说明 |
|------|------|
| **训练时** | 使用 `<PRE>` + `<SUF>` + `<MID>` 三段式拼接，让模型学习根据上下文两端预测中间 |
| **推理时** | 用户正常写代码即可，模型自动根据已写的上下文预测缺失部分，**无需用户提供 `<SUF>` 标签** |

> 训练时的三段式结构是**训练方法**，不是使用方法。推理时模型已学会"看到开头和结尾，推理中间"的能力。

---

## 🆚 普通训练 vs FIM训练

| 维度 | 普通训练 | FIM训练 |
|------|----------|----------|
| **任务本质** | 单行道续写 | 双向推理填空 |
| **上下文** | 只有上文 | 同时拥有上文和下文 |
| **模型能力** | 自由创作、写完整段落 | 精准补全、修复代码 |
| **用户场景** | 写文章开头，它续写 | 函数中间，它根据开头结尾精准补全 |

---

## ✍️ 实践技巧

| 技巧 | 说明 |
|------|------|
| **先写下期待的"结局"** | 先写断言或返回语句，模型看到结尾会倒推中间逻辑 |
| **打破循环，先写结束语** | 先打`}`或循环后的代码，再回头写循环体内部 |

---

**核心心法**：先给骨架，再让它填肉。精准、高效、恰到好处。
